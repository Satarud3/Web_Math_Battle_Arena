BLUEPRINT PROYEK WEB
MATH BATTLE ARENA
Math Battle Arena adalah web game edukasi matematika berbasis kompetisi real-time. Sistem ini mempertemukan pemain dalam mode latihan, duel 1 vs 1, turnamen, dan battle royale. Fokus utamanya adalah membuat belajar matematika terasa seperti bermain game kompetitif: ada skor, ranking, leaderboard, statistik, achievement, dan progress pemain.
Rekomendasi arsitektur utama untuk proyek ini adalah Next.js + NestJS + PostgreSQL + Socket.IO, karena cocok untuk aplikasi modern, real-time, dan scalable. Next.js mendukung App Router, layout, Server/Client Components, dan routing berbasis file; NestJS cocok untuk backend TypeScript yang rapi dan scalable; Socket.IO mendukung komunikasi event-based dua arah berlatensi rendah; Prisma cocok untuk ORM TypeScript dengan PostgreSQL. (Next.js)

1. ANALISIS MASALAH
1.1 Masalah Utama
Banyak siswa dan mahasiswa menganggap matematika sebagai pelajaran yang sulit, membosankan, dan menegangkan. Masalahnya bukan hanya pada materi, tetapi juga pada cara belajar yang sering terasa pasif, monoton, dan kurang memberi tantangan menyenangkan.
Math Battle Arena hadir untuk menyelesaikan masalah tersebut dengan mengubah proses belajar matematika menjadi pengalaman bermain game kompetitif.
1.2 Pain Point Pengguna
Pain Point | Penjelasan | Solusi Math Battle Arena
Matematika terasa membosankan | Latihan soal biasa kurang menarik | Dibuat seperti arena pertarungan
Kurang motivasi belajar | Tidak ada target yang terasa seru | Ada ranking, level, badge, leaderboard
Tidak tahu kemampuan sendiri | Siswa sulit mengukur progress | Disediakan statistik akurasi, win rate, streak
Belajar sendirian terasa monoton | Tidak ada interaksi sosial | Ada duel real-time dan battle royale
Soal latihan kurang bervariasi | Soal sering statis | Admin dapat mengelola bank soal
Tidak ada feedback cepat | Siswa baru tahu nilai setelah selesai | Jawaban langsung dinilai real-time
1.3 Mengapa Game Kompetitif Lebih Menarik?
Metode game kompetitif lebih menarik karena pengguna tidak hanya вЂњmengerjakan soalвЂќ, tetapi juga merasa sedang bertanding. Elemen seperti skor, waktu, peringkat, badge, dan kemenangan membuat pengguna terdorong untuk mencoba lagi.
Perbedaan utamanya:
Metode Biasa | Math Battle Arena
Latihan soal statis | Duel soal real-time
Nilai akhir saja | Skor langsung berubah
Belajar sendiri | Bertanding dengan pemain lain
Tidak ada ranking | Ada leaderboard
Kurang visual | UI seperti arena game
Motivasi rendah | Ada reward, badge, dan level
1.4 Latar Belakang Akademik dan Profesional
Secara akademik, proyek ini menggabungkan konsep e-learning, gamification, dan real-time web application. Gamification digunakan untuk meningkatkan motivasi belajar melalui elemen permainan seperti poin, level, ranking, tantangan, dan achievement.
Secara profesional, proyek ini dapat dikembangkan menjadi platform edutech skala kecil. Versi MVP dapat digunakan untuk tugas kuliah, lomba inovasi pendidikan, atau startup mini. Versi lanjutannya dapat dikembangkan untuk sekolah, bimbel, komunitas matematika, atau platform latihan UTBK.

10.2 Normalisasi 3NF
Tahap | Penerapan
1NF | Semua field bernilai atomik, kecuali options yang sengaja memakai JSONB untuk pilihan jawaban
2NF | Semua field non-key bergantung penuh pada primary key
3NF | Tidak ada ketergantungan transitif; role dipisah, kategori dipisah, statistik dipisah
Catatan: options memakai JSONB karena pilihan jawaban bersifat semi-terstruktur. PostgreSQL mendukung tipe JSONB dan indexing, sehingga cocok untuk data opsi soal yang fleksibel. (PostgreSQL)

16. BANGUN DATABASE
16.1 SQL Script Awal PostgreSQL
CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TABLE roles (
    id SERIAL PRIMARY KEY,
    name VARCHAR(50) UNIQUE NOT NULL,
    description TEXT
);

CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    role_id INT NOT NULL REFERENCES roles(id),
    name VARCHAR(100) NOT NULL,
    username VARCHAR(50) UNIQUE NOT NULL,
    email VARCHAR(150) UNIQUE NOT NULL,
    password_hash TEXT NOT NULL,
    avatar_url TEXT,
    is_active BOOLEAN DEFAULT TRUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE question_categories (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name VARCHAR(100) UNIQUE NOT NULL,
    description TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE questions (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    category_id UUID REFERENCES question_categories(id),
    question_text TEXT NOT NULL,
    options JSONB NOT NULL,
    correct_answer VARCHAR(10) NOT NULL,
    explanation TEXT,
    difficulty VARCHAR(20) NOT NULL CHECK (difficulty IN ('easy', 'medium', 'hard')),
    base_score INT DEFAULT 100,
    is_active BOOLEAN DEFAULT TRUE,
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE matches (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    mode VARCHAR(30) NOT NULL CHECK (mode IN ('practice', 'duel', 'tournament', 'battle_royale')),
    status VARCHAR(30) NOT NULL DEFAULT 'waiting',
    total_questions INT DEFAULT 10,
    winner_user_id UUID REFERENCES users(id),
    started_at TIMESTAMP,
    ended_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE battle_rooms (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    match_id UUID UNIQUE NOT NULL REFERENCES matches(id) ON DELETE CASCADE,
    room_code VARCHAR(30) UNIQUE NOT NULL,
    max_players INT DEFAULT 2,
    status VARCHAR(30) DEFAULT 'waiting',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE match_players (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    match_id UUID NOT NULL REFERENCES matches(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    total_score INT DEFAULT 0,
    correct_count INT DEFAULT 0,
    wrong_count INT DEFAULT 0,
    avg_answer_time NUMERIC(10,2) DEFAULT 0,
    result VARCHAR(20),
    joined_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(match_id, user_id)
);

CREATE TABLE answers (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    match_id UUID NOT NULL REFERENCES matches(id) ON DELETE CASCADE,
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    question_id UUID NOT NULL REFERENCES questions(id),
    selected_answer VARCHAR(10) NOT NULL,
    is_correct BOOLEAN NOT NULL,
    answer_time_ms INT NOT NULL,
    score_earned INT DEFAULT 0,
    submitted_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(match_id, user_id, question_id)
);

CREATE TABLE rankings (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID UNIQUE NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    rating_point INT DEFAULT 1000,
    tier VARCHAR(30) DEFAULT 'Bronze',
    current_rank INT,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE leaderboard_logs (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    ranking_id UUID NOT NULL REFERENCES rankings(id) ON DELETE CASCADE,
    match_id UUID REFERENCES matches(id) ON DELETE SET NULL,
    old_point INT NOT NULL,
    new_point INT NOT NULL,
    change_point INT NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE achievements (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    name VARCHAR(100) UNIQUE NOT NULL,
    description TEXT,
    icon_url TEXT,
    condition_type VARCHAR(50) NOT NULL,
    condition_value INT NOT NULL,
    reward_point INT DEFAULT 0,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE TABLE user_achievements (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    achievement_id UUID NOT NULL REFERENCES achievements(id) ON DELETE CASCADE,
    earned_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(user_id, achievement_id)
);

CREATE TABLE user_stats (
    id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
    user_id UUID UNIQUE NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    total_matches INT DEFAULT 0,
    total_wins INT DEFAULT 0,
    total_losses INT DEFAULT 0,
    total_draws INT DEFAULT 0,
    total_questions_answered INT DEFAULT 0,
    total_correct_answers INT DEFAULT 0,
    accuracy NUMERIC(5,2) DEFAULT 0,
    win_rate NUMERIC(5,2) DEFAULT 0,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_users_role_id ON users(role_id);
CREATE INDEX idx_questions_category ON questions(category_id);
CREATE INDEX idx_questions_difficulty ON questions(difficulty);
CREATE INDEX idx_matches_mode_status ON matches(mode, status);
CREATE INDEX idx_match_players_match ON match_players(match_id);
CREATE INDEX idx_answers_match_user ON answers(match_id, user_id);
CREATE INDEX idx_rankings_point ON rankings(rating_point DESC);
CREATE INDEX idx_user_stats_winrate ON user_stats(win_rate DESC);

17.5 Logic Matchmaking
1. Player mengirim event join_queue.
2. Server mengecek token.
3. Server membaca ranking point player.
4. Player dimasukkan ke queue sesuai mode.
5. Server mencari lawan dengan range ranking mirip.
6. Jika ditemukan, server membuat match dan battle_room.
7. Server memilih soal random sesuai difficulty.
8. Server mengirim event match_found.
17.6 Logic Scoring
Contoh scoring:
base_score = 100
time_bonus = sisa_waktu_detik * 2
streak_bonus = streak >= 3 ? 30 : 0
difficulty_bonus:
  easy = 0
  medium = 30
  hard = 60

total = base_score + time_bonus + streak_bonus + difficulty_bonus
Jika salah:
score = 0
Atau untuk mode kompetitif:
score = -10
